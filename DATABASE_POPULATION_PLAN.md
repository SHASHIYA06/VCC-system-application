## Population Strategy

### Phase 1: Immediate Population (High Priority)

#### 1. Equipment Table Population
**Source**: Extract from existing Device and Connector data
**Approach**:
- Identify unique equipment from device records
- Create equipment catalog with:
  - Unique codes
  - Names and descriptions
  - Manufacturer information
  - Part numbers
  - Subsystem associations

**SQL Template**:
```sql
INSERT INTO "Equipment" (id, name, code, subsystemId, manufacturer, model, partNumber, description, metadata)
SELECT 
  gen_random_uuid() as id,
  device_name as name,
  UPPER(REPLACE(device_name, ' ', '_')) as code,
  subsystem_id as subsystemId,
  'Unknown' as manufacturer,
  NULL as model,
  NULL as partNumber,
  'Auto-generated from device data' as description,
  '{}' as metadata
FROM existing_device_data
WHERE device_name IS NOT NULL
GROUP BY device_name, subsystem_id;
```

#### 3. DrawingVerificationStatus Creation
**Source**: Existing Drawing data
**Approach**:
- Create verification records for all 272 drawings
- Set initial status based on sync status
- Calculate confidence scores based on data completeness

**SQL Template**:
```sql
INSERT INTO "DrawingVerificationStatus" (
  id, drawingId, drawingNumber, status, pagesVerified, totalPages, confidence, extra
)
SELECT 
  gen_random_uuid() as id,
  d.id as drawingId,
  d."drawingNo" as drawingNumber,
  CASE WHEN d."isSynced" THEN 'VERIFIED' ELSE 'UNVERIFIED' END as status,
  COALESCE(d."totalSheets", 1) as pagesVerified,
  COALESCE(d."totalSheets", 1) as totalPages,
  CASE WHEN d."isSynced" THEN 0.95 ELSE 0.75 END as confidence,
  jsonb_build_object('autoGenerated', true, 'source', 'migration') as extra
FROM "Drawing" d
WHERE NOT EXISTS (
  SELECT 1 FROM "DrawingVerificationStatus" dvs WHERE dvs."drawingId" = d.id
);
```

### Phase 2: Data Enhancement (Medium Priority)

#### 4. DeviceSpecification Population
**Source**: Existing Device data with enhanced attributes
**Approach**:
- Extract specifications from device records
- Create categorized specifications
- Link to devices with verification status

**SQL Template**:
```sql
INSERT INTO "DeviceSpecification" (
  id, deviceId, specCode, specName, specValue, unit, category, verified, source, extra
)
SELECT 
  gen_random_uuid() as id,
  d.id as deviceId,
  'DESCRIPTION' as specCode,
  'Description' as specName,
  d."description" as specValue,
  NULL as unit,
  'General' as category,
  true as verified,
  'Migration' as source,
  jsonb_build_object('autoGenerated', true) as extra
FROM "Device" d
WHERE d."description" IS NOT NULL
AND NOT EXISTS (
  SELECT 1 FROM "DeviceSpecification" ds 
  WHERE ds."deviceId" = d.id AND ds."specCode" = 'DESCRIPTION'
);
```

#### 5. Cable Specification Population
**Source**: Wire data with cable information
**Approach**:
- Identify cables from wire records with cableSpec field
- Create unique cable entries
- Add technical specifications

**SQL Template**:
```sql
INSERT INTO "Cable" (
  id, cableNumber, cableType, coreCount, manufacturer, partNumber, 
  description, metadata, status
)
SELECT 
  gen_random_uuid() as id,
  DISTINCT w."cableSpec" as cableNumber,
  'MULTI_CORE' as cableType,
  NULL as coreCount,
  'Unknown' as manufacturer,
  NULL as partNumber,
  CONCAT('Auto-generated from wire data: ', w."cableSpec") as description,
  jsonb_build_object('autoGenerated', true, 'sourceCount', COUNT(*)) as metadata,
  'active' as status
FROM "Wire" w
WHERE w."cableSpec" IS NOT NULL
GROUP BY w."cableSpec";
```
