## 📝 Documentation Standards

### Active Documentation Must:
1. Be updated with each major change
2. Contain accurate, current information
3. Include "Last Updated" date
4. Reference only existing files/features
5. Provide actionable instructions

### When to Archive:
1. Information is outdated (>30 days old)
2. Feature has been completed/replaced
3. Status report is superseded by newer report
4. Build/deployment guide for old version
5. Temporary action items that are complete

## ✏️ Contributing to Documentation

### Creating New Documentation

1. Use clear, descriptive titles
2. Include "Last Updated" date at top
3. Add to "Current Documentation" section of this index
4. Follow markdown standards
5. Test all commands/instructions

### Updating Existing Documentation

1. Update "Last Updated" date
2. Mark changed sections with version notes
3. Remove outdated information
4. Update index if document purpose changes

### Archiving Documentation

1. Move file to `./archive/` directory
2. Update this index (move from Current to Archived)
3. Add archive date to filename (optional)
4. Keep for historical reference (don't delete)
